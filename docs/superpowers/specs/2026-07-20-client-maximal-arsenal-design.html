1배경 / 목표

잡스랩 자체 브랜드는 미니멀이지만 클라이언트는 맥시멀·화려함을 추구할 수 있다. 클라이언트 작업에 쓸 검증된 외부 리소스를 우리 디자인 소스에 붙여 빠르게 활용하는 전역 역량을 만든다 — 남의 마켓 나열이 아니라 "클라이언트가 X 원함 → 이 리소스를 이렇게" 큐레이션 + DaisyUI 토큰 변환기로 클라이언트 브랜드색을 즉시 입힌 시안.

확정 결정 · 라우팅 중복 0

시스템 정의/변경 = studio/design-tokens(tokens.json 유일 원천) · 포맷 출력(CSS/Tailwind/DaisyUI) = design-inject · 외부 맥시멀 리소스 = arsenal. DaisyUI는 별도 시스템이 아니라 tokens.json 소비 포맷 — studio와 경합 안 함.

범위 밖

잡스랩 자체 사이트 적용 · Origin UI(중복+legacy, 사용자 제외 지시) · 외부 서비스 자동설치·전면 래핑 · 미리보기 UI·자동 fetch(v2).

3모듈별 설계

3.1 DaisyUI 출력 포맷 design-check 실코드

arsenal이 아니라 토큰 파이프라인에. tokens-to-code.mjs에 --format daisyui 추가(기존 css/tailwind와 동일 패턴) → .design/tokens.daisyui.css(@plugin daisyui/theme 블록). 순수·결정적. design-check 1.3.0→1.4.0.

tokens-v1DaisyUI 변수
color.primary--color-primary
color.bg / surface--color-base-100 / base-200
color.text / muted--color-base-content / neutral
radius.base / card--radius-field·selector / box
secondary·accent·semanticDaisyUI 기본 상속(강제 매핑 안 함 — 오배치 방지)

함수: daisyTheme(tokens). 출력은 @plugin "daisyui/theme" 블록. all 포맷에 daisyui 포함.

3.2 리소스 레지스트리 design-studio 데이터

큐레이션 JSON skills/arsenal/registry.json — 각 리소스에 forWhat / pull / stackFit / brandNote / maximalTier. 대상: casberry·originkit·21st-magic·daisyui(pull = design-inject --format daisyui 포인터). Origin UI 제외.

3.3 arsenal 스킬 워크플로

경로 skills/arsenal/SKILL.md. 니즈 매칭 + 리소스 제시 + self 경고. 토큰 정의·변경 안 함(studio/design-tokens 몫). DaisyUI는 "design-inject로 뽑으라"는 포인터만. 커밋 금지.

self 정책

meta.project == jobslab 감지 시 경고(차단 아님) — "맥시멀은 클라이언트용, 자체 사이트엔 말림". 참고 조회는 허용.

DaisyUI 주의

DaisyUI .btn은 잡스랩 자체 DS .btn과 충돌 — 클라이언트 시안에 격리 사용, 한 페이지에 두 시맨틱 레이어 혼용 금지.

4사용자 플로우

니즈 수집→ self 경고(해당 시)→ 레지스트리 매칭→ 리소스+사용법 제시→ DaisyUI면 토큰→테마 변환→ 시안
