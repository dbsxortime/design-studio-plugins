# 브랜드 조건부 생성 (visual-studio / `generate`) — 설계

- 날짜: 2026-07-20
- 상태: 확정 (구현 계획 진행)
- 관련: design-studio(`motion`·`studio`), design-check(`design-tokens`·uipro), brand-studio(OG 파이프라인)

## 1. 문제 / 목표

이미지·영상 생성툴(Flux·SDXL·Runway·Kling·Veo 등)을 **우리 디자인 소스에 붙여서**, 클라이언트 브랜드에 맞는 결과가 나오게 한다. 단순히 "생성 API를 호출"하는 게 아니라, **클라이언트의 `.design` 토큰·브랜드·스타일 가드레일을 생성 프롬프트에 주입하고 결과를 팔레트로 후처리**하는 "브랜드 조건부 생성"이 핵심 차별점이다.

**성공 기준**: 클라이언트 프로젝트에서 "히어로 배경 만들어줘" 수준의 요청에, 그 클라이언트 팔레트·스타일·금지항목이 반영된 이미지/영상이 나오고, 출처(프롬프트·모델·seed)가 기록돼 재현·반복이 가능하다.

## 2. 비목표 (Non-goals)

- **잡스랩 자체 사이트에는 붙이지 않는다** — 잡스랩 브랜드는 "장식 0·이미지 최소"라 생성 이미지가 어울리지 않는다. **클라이언트 프로젝트 전용.**
- 커스텀 모델 학습(fine-tune)·LoRA 훈련.
- 대량 배치 생성·생성물 갤러리 UI.
- 개별 모델 MCP를 여러 개 직접 관리 — 애그리게이터 MCP 하나로 흡수.

## 3. 범위 결정 (확정)

| 축 | 결정 |
|----|------|
| 사용처 | 클라이언트 프로젝트 전용 |
| Transport | **MCP 우선** — 애그리게이터 MCP(Replicate·Fal 계열, 이미지+영상 다모델) 권장. API 어댑터가 나중에 슬롯인되게 설계. |
| 매체 | 이미지·영상 둘 다(호출 시점에 모델 선택). |
| 후처리(팔레트 락) | **v1 포함** — 이미지 전체 적용. 영상 프레임 리컬러는 v2(품질 저하 리스크). |
| 매체 v1 | 이미지 + **영상 생성 포함**(영상은 생성만, 후처리 없음). |
| 배치 위치 | **신규 플러그인 `visual-studio`**(외부 비용·의존 격리, 선택 설치) — 확정. |
| 기본 애그리게이터 | **Replicate**(넓은 이미지+영상 모델 카탈로그) — 설치 시 Fal로 교체 가능. |

## 4. 아키텍처 (유닛 4개 — 관심사 분리)

### 4.1 브랜드 조건부 코어 (`lib/prompt-brand.mjs`) — 매체·제공자 무관, 핵심 자산
입력: 클라이언트 `.design/tokens.json` + `.design/brand.json` + (선택) uipro `style-guardrails`.
출력: **프롬프트 스캐폴드** 객체
- `positive`: 사용자 의도 + 스타일 키워드(uipro 스타일/무드) + 팔레트 hex 나열 + 톤 서술.
- `negative`: 브랜드 금지항목에서 파생 — 항상 "text, watermark, logo, signature" + 브랜드가 금하면 "gradient, decorative pattern, emoji" 등.
- `aspect`: 용도→종횡비 맵(OG 1200×630, hero 16:9, square, story 9:16 …).
- `paletteLock`: 락 강도 힌트(배경·텍스처=strong, 실사 히어로=light/off).
- 이 유닛은 순수 함수 — 제공자·매체 무관하게 재사용, 단독 테스트 가능.

### 4.2 Transport (MCP, 제공자 무관)
- 런타임에 붙어 있는 생성 MCP의 도구를 **탐색(ToolSearch)** 해 호출. 애그리게이터면 모델 id를 파라미터로 넘겨 이미지/영상 선택.
- 생성 MCP가 없으면 **graceful**: "생성 MCP(예: Replicate)를 추가하세요" 안내 + 어댑터 경로 제시. 실패로 죽지 않음.
- 어댑터 인터페이스는 `generate({ prompt, negative, aspect, model, seed }) → { assetPath, meta }` 한 형태 — MCP든 (나중의) 직접 API든 같은 시그니처.

### 4.3 후처리 / 팔레트 락 (`lib/palette-lock.mjs`) — 이미지 전용(v1)
- 방식: 생성 이미지를 클라이언트 팔레트로 **양자화/듀오톤/그레이딩**하여 브랜드 톤에 수렴. 강도는 4.1의 `paletteLock` 힌트로 결정(배경=강, 실사=약/off).
- 표준 라이브러리·경량 이미지 처리로 근사(외부 무거운 의존 회피). 원본과 락본을 **둘 다 보존**해 사용자가 고른다.
- 영상 리컬러는 v1 제외(프레임별 처리 비용·품질 저하) — v2 후보.

### 4.4 배치 + 출처 기록
- 결과를 클라이언트 프로젝트 `assets/generated/`에 저장(원본 + 락본).
- **출처 기록**(`assets/generated/_provenance.json`): 프롬프트·negative·모델·seed·종횡비·비용·타임스탬프 → 재현·반복·감사 가능. motion 스킬 `add-use` 철학과 동일.

## 5. 워크플로 (`generate` 스킬, 게이트 포함)

1. **의도 수집** — 무엇을(히어로/배경/텍스처/썸네일/영상), 용도(→종횡비), 몇 장.
2. **브랜드 확보** — `.design` 없으면 `design-tokens`로 클라이언트 사이트에서 토큰 추출(기존 루프 재사용).
3. **프롬프트 조립** — 4.1로 스캐폴드 생성, 사용자에게 **프롬프트 + 예상 비용 표시**.
4. **비용 게이트** — 승인받은 뒤에만 생성(호출당 비용 상한).
5. **생성** — 4.2 MCP 호출.
6. **후처리** — 4.3 팔레트 락(이미지), 원본·락본 미리보기(스크린샷/인라인).
7. **반복** — 프롬프트·seed 조정 재생성.
8. **배치 + 기록** — 4.4.

## 6. 가드레일

- **브랜드 NEVER 리스트 → 네거티브 프롬프트 자동 반영**(이모지·그라디언트·텍스트 등).
- **비용 상한**(호출당) + 생성 전 예상 비용 고지 — 무단 대량 호출 방지.
- **seed 기록**으로 재현 보장.
- **잡스랩 self 프로젝트 감지 시 경고/차단** — 이 스킬은 클라이언트 전용.
- 생성물은 커밋하지 않음 — 변경 목록만 제시, 커밋은 사용자 위임(플러그인 공통 철칙).

## 7. 기존 자산과의 관계

- `design-tokens`(브랜드 확보) · uipro `style-guardrails`(스타일 키워드) **재사용**.
- `motion`(코드 애니메이션)과 형제 — `generate`는 픽셀/영상.
- brand-studio OG 파이프라인이 배경 생성에 이 스킬을 **선택적으로** 호출 가능.

## 8. v1 vs 이후

- **v1**: 이미지 생성 + 팔레트 락(전체) + 영상 생성(후처리 없이) + 출처 기록 + 비용 게이트.
- **v2 후보**: 영상 프레임 리컬러, 직접 API 어댑터(MCP 폴백), 배치, 참조 이미지 조건부(img2img).

## 9. 확정된 결정 (2026-07-20 검토 통과)

1. **배치**: 신규 플러그인 `visual-studio`.
2. **기본 애그리게이터 MCP**: Replicate(설치 시 Fal로 교체 가능).
3. **영상 v1**: 생성 포함(후처리 없음 — 리컬러는 v2).
