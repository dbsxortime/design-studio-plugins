브랜드 조건부 생성 — visual-studio / generate

작성일 2026-07-20 대상 클라이언트 작업 전용 상태 검토 대기

1배경 / 목표

이미지·영상 생성툴(Flux·SDXL·Runway·Kling·Veo 등)을 우리 디자인 소스에 붙여, 클라이언트 브랜드에 맞는 결과가 나오게 한다. 단순 API 호출이 아니라 클라이언트의 토큰·브랜드·스타일 가드레일을 프롬프트에 주입하고 결과를 팔레트로 후처리하는 "브랜드 조건부 생성"이 차별점이다.

확정 결정

사용처 = 클라이언트 전용 · Transport = MCP 우선(애그리게이터 권장) · 매체 = 이미지·영상(호출 시점 선택) · 팔레트 락 v1 포함(이미지 전체).

범위 밖

잡스랩 자체 사이트 적용(브랜드 미니멀 유지) · 커스텀 모델 학습 · 대량 배치 · 생성물 갤러리 UI.

2현황

항목현 상태
실사 이미지/영상 생성 MCP미연결 새로 붙여야 함
Figma MCP디자인·벡터 생성 + Figma 내부 모션 export 위주(실사 생성 아님)
image-archive MCP생성이 아닌 검색·보관용
브랜드 확보 경로있음 design-tokens 스킬이 클라이언트 사이트에서 토큰 추출
스타일 지식있음 uipro style-guardrails(스타일·무드 키워드)

3모듈별 설계

3.1 브랜드 조건부 코어 핵심 자산

매체·제공자 무관 순수 함수. 클라이언트 .design/tokens.json + .design/brand.json + uipro 가드레일 → 프롬프트 스캐폴드 생성.

출력내용
positive의도 + 스타일·무드 키워드 + 팔레트 hex + 톤 서술
negative브랜드 금지항목 파생 — 항상 text·watermark·logo, 금지 시 gradient·pattern·emoji
aspect용도→종횡비(OG 1200×630 · hero 16:9 · square · story 9:16)
paletteLock락 강도 힌트(배경·텍스처=강, 실사 히어로=약/off)

3.2 Transport (MCP, 제공자 무관)

붙어 있는 생성 MCP 도구를 런타임 탐색해 호출. 애그리게이터면 모델 id로 이미지/영상 선택. 어댑터 시그니처 하나(generate(prompt, negative, aspect, model, seed) → asset, meta)로 MCP·훗날 직접 API를 동일 취급.

주의

생성 MCP 미연결 시 graceful — "이 MCP를 추가하라" 안내로 처리, 실패로 죽지 않음.

3.3 후처리 / 팔레트 락 v1 포함

생성 이미지를 클라이언트 팔레트로 양자화·듀오톤·그레이딩해 브랜드 톤에 수렴. 강도는 3.1 힌트로 결정. 원본·락본 둘 다 보존해 사용자가 고른다. 영상 리컬러는 v2(품질 저하 리스크).

3.4 배치 + 출처 기록

결과를 assets/generated/에 저장, assets/generated/_provenance.json에 프롬프트·모델·seed·비용 기록 → 재현·반복·감사. motion 스킬 사용이력 철학과 동일.

4사용자 플로우

의도 수집→ 브랜드 확보→ 프롬프트 조립→ 비용 게이트→ 생성(MCP)→ 팔레트 락→ 미리보기·반복→ 배치 + 기록
가드레일

브랜드 NEVER 리스트 → 네거티브 자동 반영 · 호출당 비용 상한 + 생성 전 비용 고지 · seed 기록으로 재현 · 잡스랩 self 프로젝트 감지 시 경고 · 생성물 커밋 금지.

5의사결정 로그

결정선택근거
사용처클라이언트 전용잡스랩 자체 브랜드는 장식 0·이미지 최소 — 생성물 부적합
매체 고정?고정 안 함작업 단계가 매번 달라 이미지/영상 고정은 경직 — 애그리게이터로 호출 시점 선택
TransportMCP 우선MCP 중심 환경 + 기존 플러그인/스킬 구조와 결이 같음, 키 관리 없음
팔레트 락v1 전체 포함사용자 요청 — 브랜드 조건부의 핵심이므로 뒤로 미루지 않음
배치 위치신규 플러그인 권장 검토외부 비용·의존 격리, 선택 설치 — design-studio 내 스킬도 대안
