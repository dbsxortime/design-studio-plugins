# 폰트 목록

> **이 목록 외 폰트 제안 금지.** /brand 미팅에서 사용자에게 폰트를 제안할 때는 반드시 아래 표 안에서만 고른다. 이유 — (1) 전부 무료·재배포 가능 라이선스(SIL OFL / Apache 2.0)라 상업 프로젝트에서 웹폰트로 안전하게 쓸 수 있고, (2) 한글 지원 여부가 검증돼 있어 "막상 한글이 깨지는" 사고를 막는다. 사유(상용) 폰트(예: 유료 그로테스크·지오메트릭 계열)는 라이선스 위반 위험이 있어 이름조차 꺼내지 않는다.

폰트 제안 순서: 타이포형 질문 트리(logo-playbook 2절)에서 정한 **느낌 4분류**를 이 표의 '느낌 분류' 열과 대조해 후보를 좁힌 뒤, 한글이 필요한 프로젝트면 '한글 지원 = 지원' 행만 제시한다.

## 느낌 4분류 매핑 요약

- **세리프**(신뢰·전통·격조): Noto Serif KR · 나눔명조 · 고운바탕 · Lora
- **기하 산세리프**(현대·명료·중립): Pretendard · Noto Sans KR · IBM Plex Sans KR · 나눔스퀘어 네오 · 나눔고딕 · Inter · Roboto · Space Grotesk
- **라운드**(친근·부드러움): 고운돋움 · Nunito · Quicksand
- **모노**(테크·정밀·코드): JetBrains Mono · IBM Plex Mono · Space Mono

왜 이 4분류인가 — 폰트의 인상은 세부 형태보다 이 4갈래에서 먼저 갈린다. 사용자가 폰트 이름을 몰라도 "신뢰감 있게 / 현대적으로 / 친근하게 / 테크하게" 중 하나는 고를 수 있고, 그 답이 곧 분류가 된다. 한글 프로젝트의 안전한 기본값은 Pretendard(기하 산세리프·전 굵기 지원)다.
