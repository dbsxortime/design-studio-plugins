# Design Studio 열기

1. 대상 프로젝트 루트에서 `.design/tokens.json` 존재를 확인한다.
   - 없으면: "기준 토큰이 없습니다 — /design-tokens init을 먼저 실행하세요(design-check 플러그인)"라고 안내하되, 사용자가 그냥 열길 원하면 토큰 없이 진행(주입 없음, 열람만).
2. 서버 실행 (이 스킬 파일 기준 플러그인 루트 = `../..`):
   `node <plugin-root>/scripts/studio-serve.mjs --tokens <project>/.design/tokens.json`
   - 이미 :28572에 스튜디오가 떠 있으면 스크립트가 재사용하고 브라우저만 연다.
   - 포트를 다른 프로세스가 점유한 경우 스크립트가 에러를 내면 그 메시지를 그대로 사용자에게 전달.
3. 사용자에게 안내: 조합기 좌측 소스 목록 맨 위 "사용자 토큰"이 현재 기준이며,
   속성별로 다른 시스템과 믹스한 뒤 우측 상단 "기준 내보내기"를 누르면
   `.design/tokens.json`이 갱신된다.
4. 사용자가 편집을 마쳤다고 하면 tokens.json을 다시 읽어 변경 요약(무엇이 바뀌었는지)을 보고한다.

주의: 서버는 30분 유휴 시 자동 종료된다. 세션을 끝낼 때 서버를 강제 종료할 필요는 없다.
